# BDD with behave (optional)

Estimated time: 2 hours

## Definition

Understand when behavior-focused specifications help communication and when they become costly ceremony.

## What You Practice

- Given-when-then
- Ubiquitous language
- Feature files
- Acceptance criteria

## Practical Applications

- Cross-team requirements
- acceptance tests
- shared domain language

## Example Focus

The runnable example in `examples/example_basic.py` shows one concrete way to apply bdd with behave (optional) in a small, self-contained scenario.

## Why It Matters

This topic helps you decide what kind of feedback you need from a test and how to keep that feedback trustworthy as the codebase grows.

## Practice Task

Goal: Implement a small shopping cart domain that can later be described in Gherkin.

Success criteria:
- Implement `ShoppingCart.add_item`.
- Implement `ShoppingCart.total`.
- Implement `ShoppingCart.clear`.
