# TDD Basics

Estimated time: 2-3 hours

## Definition

Practice the red-green-refactor loop with a problem small enough to iterate on quickly.

## What You Practice

- Red-green-refactor
- Small increments
- Refactoring safety
- Specification by example

## Practical Applications

- Parsing helpers
- business rules
- kata-style learning

## Example Focus

The runnable example in `examples/example_basic.py` shows one concrete way to apply tdd basics in a small, self-contained scenario.

## Why It Matters

This topic helps you decide what kind of feedback you need from a test and how to keep that feedback trustworthy as the codebase grows.

## Practice Task

Goal: Implement a classic TDD kata with incremental requirements.

Success criteria:
- Implement `add(expression)`.
- Support empty strings and comma-separated integers.
- Raise `ValueError` when negative numbers are present.
